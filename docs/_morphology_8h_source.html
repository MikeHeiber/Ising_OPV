351  // This function calculates the additional change in energy of the system that would occur if the adjacent sites at (x1,y1,z1) and (x2,y2,z2) were to be swapped with
352  // a given preferential domain growth direction. This additional energy is to be used to modify the total energy change from swapping the two sites.
353  // Sites must be adjacent to each other for calculation to be correct. (Works for adjacent sites across periodic boundaries)
354  // When non-periodic/hard z-boundaries are used, it is assumed that neither site type has a preferential interaction with the z-boundary
355  // The values for growth_direction are 1 for x-direction, 2 for y-direction, and 3 for z-direction adjustment.

358  // This function calculates the anisotropy of the domains based on the directionally-dependent pair-pair correlation functions
359  // The correlation function is calculated from each starting site out to the cutoff distance.
360  // The correlation length in each direction is defined as the distance at which the pair-pair correlation function first crosses the value equal to the mixing fraction
361  // If this cross-over point is not reach within the cutoff distance, the function generates an error message and returns -1.
362  // For large lattices, the correlation function does not need to be calculated starting from every site to collect enough statistics and instead a sampling of starting sites can be used.
363  // When the total number of sites is greater than N_sampling_max, N_sampling_max sites are randomly selected and saved for performing a correlation function calculation by sampling.
364  // When the total number of sites is less than N_sampling_max, all sites will be used as starting points for the correlation function calculation.
365  bool calculateAnisotropy(const std::vector<long int>& correlation_sites, const char site_type, const int cutoff_distance);
366 
367  // This function calculates the domain size of the morphology based on the pair-pair correlation function
368  // The correlation function is calculated from each starting site out to the cutoff distance.
369  // The domain size is defined as the distance at which the pair-pair correlation function first crosses the value equal to the mixing fraction
370  // If this cross-over point is not reach within the cutoff distance, the function returns false.
371  // When the extended calculation is enabled the correlation function must reach the next peak, otherwise the function returns false.
372  // For large lattices, the correlation function does not need to be calculated starting from every site to collect enough statistics and instead a sampling of starting sites can be used.
373  // When the total number of sites is greater than N_sampling_max, N_sampling_max sites are randomly selected and saved for performing a correlation function calculation by sampling.
374  // When the total number of sites is less than N_sampling_max, all sites will be used as starting points for the correlation function calculation.
375  // If the function returns false and the function is re-called with a larger cutoff_distance, the correlation function is not recalculated for close distances and only fills in the missing data for larger distances.
376  double calculateCorrelationDistance(const std::vector<long int>& correlation_sites, std::vector<double>& correlation_data, const double mix_fraction, const int cutoff_distance, const CorrelationCalc_Params& params);
